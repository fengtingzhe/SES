# 《看海去》Game Design

> 本文档面向游戏策划 / 制作人，用于记录当前已经确认的玩法设计。
>
> 它不替代 `README.md`、`GPT_DEMO/README.md` 或 `AI_TASKS/`，而是作为策划理解与维护玩法结构的主文档。

---

## 1. 游戏一句话

《看海去》是一款以“远征探索”为核心的 2D 45 度俯视角轻策略游戏。

玩家出生在被森林包围的部落中，从未见过海。玩家需要带领部落工人，收集并分配辉石，开辟道路，点亮营地，穿越森林，最终抵达海边。

---

## 2. 核心体验

游戏的核心体验不是硬核战斗，而是：

```text
探索未知
↓
收集辉石
↓
派遣工人开路
↓
建设安全营地
↓
承担夜晚风险
↓
继续向远方推进
```

玩家体验重点：

```text
远征感
路线规划
资源取舍
工人风险
营地经营
神秘森林氛围
```

## 3. 当前 Demo 目标

当前 `GPT_DEMO` 阶段用于快速验证玩法方向，不追求工程质量。

当前 Demo 重点验证：

```text
移动 + 互动
探索 + 开路
辉石资源分配
工人派遣与风险
夜晚怪物压力
固定营火安全点
营地矿山经营雏形
```

暂时不验证：

```text
剧情演出
奇遇系统
职业系统
复杂建筑
正式美术资源
正式关卡节奏
移动端实现
```

## 4. 世界观与主题

玩家来自一个被森林完全包围的部落。

部落中的人没有真正见过海，只从历代记录和传说中知道“海”的概念。

主角从小产生了去看海的梦想。游戏的终极目标是：

```text
从森林部落出发，穿越未知世界，抵达海边。
```

世界基调：

```text
神秘
孤独
诗意
未知
远征
微光对抗黑暗
```

---

## 5. 美术与表现方向

当前方向：

```text
2D
45 度俯视角 / 等距视角
国际化 Steam 向风格
低饱和神秘森林
日夜交替
迷雾与微光
诗意但不国产化
```

不采用 3D，原因：

```text
独立开发成本更低
更适合未来移动端
更容易控制美术规模
更符合诗意远征氛围
```

---

## 6. 操作设计

### 6.1 PC 操作

当前确认的 PC 操作原则：

```text
移动 + 互动
```

具体操作：

```text
WASD / 方向键：移动主角
Space：与附近目标互动
R：重新开始 / 重新生成 Demo
```

设计目标：

```text
降低操作复杂度
接近 Kingdom New Lands 的极简操作体验
让策略深度来自决策，而不是复杂按键
```

### 6.2 Space 互动逻辑

玩家需要靠近目标后按 Space 互动。

可互动对象包括：

```text
辉石簇
散落辉石
树障
断桥
旧火塘
矿山
蓝色信标
```

Space 会自动选择附近最近或优先级最高的目标。

### 6.3 移动端方向，尚未开发

未来移动端方向：

```text
横屏
左侧虚拟摇杆：控制上下左右移动
右侧点击 / 按钮：互动
```

移动端暂时只记录方向，后续再开发。

## 7. 核心循环

### 7.1 远征循环

```text
探索未知区域
↓
发现辉石 / 树障 / 断桥 / 旧火塘
↓
消耗辉石派工人开路
↓
点亮新营地
↓
扩展安全范围
↓
继续向海推进
```

### 7.2 夜晚压力循环

```text
夜晚降临
↓
安全区外出现黑林影
↓
黑林影攻击玩家 / 工人
↓
辉石掉落并可能被怪物抢走
↓
玩家承担资源损失或人员损失
```

### 7.3 营地经营循环

```text
点亮旧火塘
↓
形成营地
↓
营地拥有矿山
↓
派驻工人采矿
↓
矿山周期性产出辉石
↓
玩家回收辉石
↓
支持下一次远征
```

## 8. 资源：辉石

辉石是当前唯一核心资源。

辉石承担多重作用：

```text
开路资源
工人任务成本
玩家生命缓冲
怪物抢夺目标
营地经营产出
远征推进燃料
```

### 8.1 辉石价值规则

当前确认规则：

```text
一颗辉石 = 1
两颗辉石组成的辉石簇 = 2
```

表现规则：

| 类型 | 视觉 | 拾取收益 |
|---|---|---:|
| 单颗散落辉石 | 1 颗小晶体 | +1 |
| 自然辉石簇 | 2 颗小晶体 | +2 |

### 8.2 地上辉石规则

当前方向：

```text
地上的辉石不自动形成保护圈。
玩家可以捡。
怪物也可以捡。
```

设计原因：

```text
避免玩家通过高频“丢辉石 / 捡辉石 / 再丢辉石”进行临场操作救场。
让游戏保持事前规划型轻策略。
```

---

## 9. 工人系统

工人是玩家推进世界的主要执行单位。

玩家自身不直接砍树、修桥、点亮营火，而是通过辉石派遣工人执行任务。

### 9.1 工人可执行任务

当前任务包括：

```text
砍树
修桥
点亮旧火塘
采矿
返回安全点
```

### 9.2 派工规则

当前已确认：

```text
玩家靠近目标后按 Space 互动。
系统选择可调度工人中，到目标可通行路径最近的工人。
工人沿路径前往目标。
工人不会穿越森林、河流、岩石等不可通行地形。
```

可参与普通任务调度的工人：

```text
idle 空闲工人
returning 返回中的工人
```

不可参与普通任务调度的工人：

```text
moving 正在前往任务的工人
working 正在工作的工人
mining 正在矿山工作的工人
lost 失踪工人
```

### 9.3 工人完成任务后的返回

工人完成砍树、修桥、点亮营火后，会返回最近的安全点。

安全点包括：

```text
部落
已点亮营火
```

工人返回途中可以被重新调度。

## 10. 地图与探索

当前地图为随机生成。

当前方向：

```text
地图具有随机性
玩家出生点 / 路线 / 资源分布具备变化
但关键节奏点需要逐步变得可控
```

地图元素包括：

```text
部落
森林
树障
河流
断桥
旧火塘
营火
矿山
辉石簇
散落辉石
蓝色信标
```

设计矛盾：

```text
随机性可以提高新鲜感。
但情感向表达需要控制节奏。
```

当前解决方向：

```text
地图和事件可随机。
关键目标、主题递进、营地节奏点需要被设计控制。
```

---

## 11. 营火与营地

### 11.1 营火不是任意建筑

当前确认：

```text
营火不能在任意地点建造。
营火只能在地图预设的旧火塘 / 石环 / 灰烬遗址上点亮。
```

设计原因：

```text
控制游戏节奏。
避免玩家走几步就建营火导致辉石耗尽。
让安全点成为路线规划中的战略节点。
```

### 11.2 旧火塘

旧火塘是地图中的固定营地候选点。

规则：

```text
玩家发现旧火塘
↓
靠近旧火塘按 Space
↓
消耗辉石派工人点亮
↓
旧火塘变成营火 / 营地
```

### 11.3 营火作用

点亮后的营火：

```text
提供安全区
怪物不靠近
成为工人返回点
解锁营地功能
```

## 12. 矿山经营系统

矿山经营是当前下一阶段要验证的营地模拟经营玩法。

### 12.1 核心目标

验证玩家是否愿意：

```text
把一部分工人长期派驻到矿山，
用远征能力换取稳定辉石产出。
```

### 12.2 矿山生成规则

当前确认方向：

```text
每个营地拥有一个矿山。
矿山是固定点。
矿山不可任意建造。
```

营地形成后：

```text
旧火塘点亮
↓
营地形成
↓
营地旁生成 / 激活矿山
```

### 12.3 派驻矿山规则

玩家靠近矿山按 Space：

```text
如果矿山无人：派最近可调度工人去矿山采矿。
如果矿山有人：后续可考虑召回，但当前需要进一步确认。
```

当前确认：

```text
工人一旦被派驻矿山，就进入矿山工作状态。
采矿工人不再参与普通任务分配。
```

采矿工人不参与：

```text
砍树
修桥
点亮营火
最近工人计算
```

### 12.4 矿山产出规则

当前确认：

```text
每 10 秒产生 1 个辉石。
```

建议第一版表现：

```text
矿山每 10 秒产出 1 个单颗散落辉石。
玩家靠近后按 Space 拾取。
```

设计意义：

```text
夜晚玩家可以在营地附近进行低强度经营。
营地成为资源恢复点。
远征和经营形成双循环。
```

## 13. 夜晚与怪物

夜晚的主要压力来自黑林影。

### 13.1 黑林影行为方向

当前方向：

```text
黑林影攻击玩家 / 工人。
目标掉落辉石。
黑林影优先捡地上的单颗散落辉石。
黑林影捡到辉石后撤退 / 消失。
```

### 13.2 玩家受击

规则：

```text
玩家有辉石：被攻击后掉落若干辉石。
玩家无辉石：再次被攻击则游戏结束。
```

### 13.3 工人受击

当前方向：

```text
工人携带任务辉石时：被攻击后任务辉石掉落。
工人无辉石时：被攻击后失踪。
```

### 13.4 设计目的

```text
辉石不是单纯货币，而是可被世界抢走的远征资源。
夜晚风险不是直接战斗，而是资源与人员损失。
```

## 14. 胜利与失败

### 14.1 阶段胜利

当前 Demo 阶段目标：

```text
抵达蓝色信标。
```

正式游戏终极目标：

```text
抵达海边。
```

### 14.2 失败条件

当前 Demo 中：

```text
玩家辉石为 0 后，再次被怪物攻击，游戏结束。
```

工人可能：

```text
被怪物拖走 / 失踪。
```

## 15. 当前已确认规则

```text
游戏是 2D 45 度俯视角 / 等距视角。
PC 操作为 WASD / 方向键移动 + Space 互动。
移动端未来方向为左摇杆移动 + 右侧互动。
辉石是唯一核心资源。
一颗辉石 = 1。
自然辉石簇 = 2。
地上辉石不自动保护。
怪物可以抢走散落辉石。
工人通过辉石执行任务。
工人完成普通任务后返回最近安全点。
返回中的工人可以被重新调度。
采矿中的工人不参与普通任务调度。
营火只能在固定旧火塘点亮。
营火 / 部落是稳定安全点。
每个营地有一个矿山。
矿山每 10 秒产生 1 个辉石。
```

## 17. 未来扩展方向

以下内容为未来玩法拓展候选，不代表当前 Demo 已实现。

### 17.1 总原则：参考 Kingdom New Lands 的物件化设计

后续玩法拓展应尽量避免复杂面板、复杂数值和复杂系统。

优先采用类似《Kingdom New Lands》的设计方式：

```text
少 UI
少数值
少按钮
规则靠世界物件表达
玩家看到一个东西，就大概知道能做什么
```

设计原则：

```text
不要先加系统，先加物件。
每个新玩法最好对应地图上的一个明确物件。
玩家靠近物件，按 Space，消耗辉石，下达命令。
```

### 17.2 流民火堆

地图上出现流民火堆。

候选规则：

```text
玩家靠近流民火堆
↓
按 Space，给予 1 个辉石
↓
流民加入
↓
流民走回最近营地
↓
等待拿工具 / 分配角色
```

设计价值：

```text
用直观方式扩充人口。
不需要招募菜单。
为后续职业系统铺路。
```

### 17.3 工具架，而不是职业菜单

未来职业系统不优先做 UI 面板，而是用营地里的工具架表达。

候选工具架：

```text
斧头架：拿了斧头的人会砍树
矿镐架：拿了矿镐的人会采矿
灯笼架：拿了灯笼的人会守夜 / 扩大视野
```

候选规则：

```text
玩家靠近工具架
↓
按 Space，消耗辉石放置工具
↓
流民 / 空闲工人拿到工具
↓
自然转变为对应角色
```

设计价值：

```text
职业通过工具表达，不通过复杂菜单表达。
与 Kingdom New Lands 的弓、锤子逻辑接近。
```

### 17.4 木门遗址 / 栅栏

营地两侧可以有破木门或栅栏遗址。

候选规则：

```text
玩家靠近木门遗址
↓
按 Space，消耗辉石派工修复
↓
夜晚怪物先攻击木门
↓
木门损坏后，怪物才能更深入营地抢辉石
```

设计价值：

```text
提供直观防守目标。
不需要玩家直接战斗。
让营地边界更清楚。
```

### 17.5 灯笼 / 守夜人

不做传统战斗单位，优先做符合《看海去》气质的“光”机制。

候选规则：

```text
工人拿灯笼
↓
夜晚站在营地边缘
↓
黑林影靠近速度变慢
```

设计价值：

```text
用光延缓怪物，而不是用武器消灭怪物。
符合微光对抗黑暗的主题。
```

### 17.6 望台遗址

地图中可出现望台遗址。

候选规则：

```text
玩家靠近望台遗址
↓
按 Space，消耗辉石派工修复
↓
修复后扩大周围可见范围
↓
帮助玩家规划下一段远征路线
```

设计价值：

```text
强化路线规划。
让玩家通过世界物件获得信息，而不是通过复杂地图 UI。
```

### 17.7 颠倒森林

颠倒森林是一种特殊区域玩法，用于表达世界规则被扭曲的光怪陆离感。

候选规则：

```text
玩家进入颠倒森林区域
↓
移动方向反转
↓
W / 上：角色向下移动
S / 下：角色向上移动
A / 左：角色向右移动
D / 右：角色向左移动
↓
离开区域后操作恢复正常
```

设计限制：

```text
只反转移动，不反转 Space 互动。
区域体量要小。
流程要短。
目标要明确。
不适合做长时间往返或复杂资源管理。
```

视觉方向：

```text
树木颜色异常
地面偏冷色 / 紫色 / 苍白
树根朝上，枝叶下垂
迷雾更重
风向、落叶、光影都呈现违和感
```

设计价值：

```text
用极少系统成本制造强记忆点。
让旅程中出现“规则被扭曲”的特殊区域。
适合作为短小、强识别、低频率的特殊路段或奇遇区。
```

### 17.8 矿山继续保持建筑点逻辑

矿山不做复杂 UI。

长期方向仍然是：

```text
营地旁有矿山
↓
派驻一个拿矿镐的人
↓
每隔一段时间掉出一颗辉石
↓
玩家靠近后按 Space 拾取
```

设计价值：

```text
矿山是营地经营的基础物件。
它提供夜间低强度玩法，但不应变成复杂经济系统。
```

### 17.9 未来扩展候选列表

后续可继续讨论：

```text
流民招募
工具架职业表达
木门 / 栅栏防守
灯笼守夜人
望台扩大视野
颠倒森林特殊区域
奇遇系统：狐狸成亲等低认知小玩法
更多营地建筑
路线事件
天气与季节
更强的随机地图节奏控制
最终抵达海边的情感演出
```

## 18. 当前设计一句话总结

```text
《看海去》是远征规划型轻策略游戏：玩家用极简操作移动与互动，消耗并保护辉石，派遣工人开辟道路，点亮固定营地，在夜晚抵御资源被怪物抢夺的压力，并通过营地矿山经营恢复资源，最终向海推进。
```
