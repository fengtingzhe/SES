# Changelog

## v0.6-refactor

状态：临时辉石拾回修复 + 矿山与持续产出迁移完成。

内容：
- 修复玩家主动放置的临时辉石无法拾回的问题。
- 自然辉石继续靠近自动拾取，临时辉石不自动拾回，但可通过 Space 主动拾回。
- 黑影仍会把临时辉石作为最高优先级局部目标并吃掉。
- 从 `GPT_DEMO/index.html` 定向迁移矿山地块和出生点矿山生成。
- 新增采矿派工：默认 0 辉石成本，矿山 reserved / occupied 防止重复派工。
- 工人到达矿山后进入持续采矿状态，约 30 秒产出 1 个辉石并掉落在附近。
- 工人逃跑或被黑影抓走 lost 时，会释放矿山占用，避免“矿山已有工人”卡死。
- HUD 显示采矿工人数，WorldRenderer 显示矿山 / 前往矿山 / 采矿中状态。
- 本轮没有迁移围墙、弓箭手、流民、职业系统或特殊事件。

## v0.5-refactor

状态：v0.4 小修复 + 工人夜晚避险、被抓与任务释放迁移完成。

内容：
- 修正初始提示文案，不再停留在 v0.3。
- 修正黑影同场上限判断为达到上限即暂停生成。
- 修正黑影到达 raidCamp 附近后无故返回雾门的问题；夜晚中无局部目标时保持家园推进 / 停留。
- 从 `GPT_DEMO/index.html` 定向迁移工人夜晚 3 格威胁感知。
- 工人执行任务时感知黑影后会释放当前 reserved，进入撤退状态并返回所属家园。
- 撤退中的工人不可被重新派工，安全回家后恢复为空闲工人。
- 黑影局部目标新增工人，目标优先级在本轮范围内为辉石 > 工人 > 玩家 > 家园推进。
- 黑影接触工人后，工人进入 lost 状态，释放原任务 reserved，黑影消失。
- HUD 显示撤退工人数和失踪工人数，WorldRenderer 显示撤退状态并隐藏 lost 工人。
- 本轮没有迁移围墙、弓箭手、矿山、流民、职业系统或特殊事件。

## v0.4-refactor

状态：昼夜与黑影局部压力迁移完成。

内容：
- 从 `GPT_DEMO/index.html` 定向迁移一天 95 秒、白天 / 黄昏 / 夜晚阶段和夜晚开始计数重置。
- 新增雾门地块，地图中至少生成 2 个雾门，并支持黑影夜晚从随机雾门出现。
- 新增黑影系统：每夜默认最多 2 个，约 3 秒间隔，同场超过 5 个暂停生成，HP 为 1。
- 黑影出现时选择最近家园作为 raidCamp；无局部目标时向该家园推进。
- 新增 4 格方格战术感知和约 1.2 秒目标锁，避免黑影全图追逐或频繁抖动。
- 临时辉石接入诱敌：黑影优先靠近 4 格内辉石，接触后辉石移除且黑影消失。
- 玩家被黑影接触时损失 1 个辉石；0 辉石时进入失败状态，R 重置仍可用。
- 夜晚结束后黑影返回最近雾门并消失。
- HUD 显示天数、阶段、黑影数量和当夜生成计数。
- 本轮没有迁移围墙、弓箭手、矿山、流民、工人夜晚避险、工人被抓或特殊事件。

## 已废弃路线

以下旧版本路线已经废弃，不再作为主线继续开发：

- WEB_DEMO v0.2 核心循环迁移
- WEB_DEMO v0.3 GameConfig 基础配置集中化
- WEB_DEMO v0.4 夜晚黑影压力系统
- WEB_DEMO v0.5 围墙 / 弓箭手 / 防御系统
- WEB_DEMO v0.6 矿山 / 流民 / 人口补给系统

废弃原因：
- 旧路线更接近重新开发一套类似 GPT_DEMO 的功能。
- 旧路线没有先提取 GPT_DEMO 已验证的玩法规则、交互逻辑和体验节奏。
- 新路线必须先建立规则基线，再按基线工程化重构 WEB_DEMO。
