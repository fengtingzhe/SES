# Known Issues

## v1.0-refactor 分类总览

### 必须修复问题

当前没有新增 P0 / P1 阻塞问题。v1.0 的小地图、HUD、标签提示和文档同步应作为体验回归基线继续保持；若后续验证发现完整旅程无法闭环、HUD 关键状态缺失、或小地图暴露未探索信息，应升级为必须修复问题并补充任务卡。

### 策划待确认问题

- 围墙是否继续作为可通行地块：当前按 GPT_DEMO 表现保留为可通行，只作为 HP 防御点与黑影目标；若改为地形阻挡，需要标记为有意重设计。
- 黑影攻击弓箭手 / 弓箭手 lost 是否正式保留：当前实现与 GPT_DEMO 目标优先级一致，但曾相对 v0.8 任务卡超前。
- 采矿派工是否保持 0 辉石成本：当前继承 GPT_DEMO 原型表现。
- 工人夜晚威胁感知范围是否继续使用当前规则，还是与黑影 4 格战术感知统一。
- 流民火堆是否允许无限冷却刷新，是否需要次数上限或地图资源上限。
- 新营地周边职业点、墙基、矿山等设施相对位置是否需要正式关卡布局规则。
- 一天 95 秒是否锁定为正式默认节奏，还是后续开放调参。
- 狐狸婚仪和终点看海是否仅保留基础提示，还是进入正式剧情演出与结算系统。
- F3 / M 小地图、HUD 辅助信息和标签提示是否作为正式功能保留，还是后续改为调试 / 辅助开关。

### 后续版本内容

- 正式剧情演出、完整结算系统、正式图片 / 音乐 / 字体资源。
- 存档系统、CSV / JSON 配置读取、移动端摇杆或完整移动端适配。
- 围墙升级、营地损坏、营地 HP 或更多防御成长。
- 黑影攻击返程流民、流民夜晚减速、流民 lost 等流民风险扩展。
- 完整职业系统、职业升级、职业 UI 和更细的职业平衡。
- 小地图图例、标签开关、辅助信息面板等更完整的信息架构。

### 暂不处理 / 可接受限制

- 小地图当前是轻量辅助视图，只显示已探索地块和少量关键标记，不承担完整导航、图例或战术雷达功能。
- 临时辉石保持“不能自动拾回、可 Space 主动拾回”的 v0.6 规则；自然辉石保持靠近自动拾取。
- 狐狸婚仪仍是最小事件表现，没有正式镜头、音效、美术和剧情演出。
- 终点完成仍是基础胜利提示和 `completed` 状态，不是完整结算系统。
- 围墙当前可通行，直到策划确认前不改为阻挡。

## v0.8-audit-fix 待确认问题

### 黑影攻击弓箭手是否正式保留

v0.8 代码已经实现黑影把弓箭手作为局部目标，并在接触后使弓箭手进入 lost 状态。

该行为与 v0.8 原任务卡“不做黑影攻击弓箭手”的限制存在轻微范围偏差。由于 GPT_DEMO 的黑影目标优先级包含弓箭手，当前实现暂时保留为“有意超前实现 / 待策划确认”。

后续需要确认：
- 保留：正式纳入黑影目标规则。
- 移除：删除 ARCHER 目标伤害分支和 hitArcher。

### 围墙是否应该可通行

当前 WEB_DEMO 中 WALL 仍然是可通行地块。

这可能继承了 GPT_DEMO 当前表现，但从防线直觉看，围墙也可能应该阻挡玩家、工人和黑影移动。

后续需要确认：
- 继承 GPT_DEMO：围墙可通行，只作为黑影目标 / HP 防御点。
- 改为正式规则：围墙不可通行，成为真正地形阻挡；若这样做，应标记为有意重设计。
