## 已修复

### WEB_DEMO/index.html 曾是旧版单文件 Demo

```text
严重程度：高
问题：WEB_DEMO/index.html 内嵌旧版可玩 Demo 的 CSS 与 JS，没有加载 /src/main.js。
影响：npm run dev 打开的不是 Vite + src/main.js 工程骨架，docs/acceptance_tests.md 的验收标准不成立。
修复：已将 WEB_DEMO/index.html 恢复为 Vite 薄入口，只加载 /src/main.js。
备注：旧版单文件 Demo 可从 Git 历史中找回；本次未从截断 diff 中创建 legacy 备份，避免恢复不完整文件。
```

## 当前已知问题

### v0.2 仅迁移最小核心循环

```text
严重程度：中
问题：v0.2 不包含 GPT_DEMO gpt.11.7 中的黑影、围墙、弓箭手、矿山、流民火堆、狐狸奇遇、颠倒森林、移动端虚拟摇杆、正式图片资源、音乐音效、CSV 读取和存档系统。
影响：当前版本只能验证探索、辉石、工人派遣、砍树、修桥、点亮营地、日夜循环和阶段目标，不能代表完整玩法草图。
当前状态：按任务卡要求暂不实现，留到后续版本拆分迁移。
```

### 工人完成任务后的返回点仍需试玩验证

```text
严重程度：低
问题：v0.2 已实现工人完成任务后返回部落或最近已激活营地，但营地激活后的返回体验、路径选择和长期调度节奏仍需要试玩验证。
影响：如果后续地图更复杂，工人返回点可能需要更严格的最近营地规则和路径可达性提示。
当前状态：后续版本继续验证。
备注：工人完成任务后是否应返回最近营地，后续版本继续验证。
```

### 地图节奏仍是代码内生成

```text
严重程度：低
问题：v0.2.1 已为地图加入轻随机，但地图结构、段落区间和数值仍在代码中生成，没有接入 CSV / JSON 配置。
影响：策划调参仍需改代码。
当前状态：v0.3 已将地图生成参数集中到 GameConfig.js；未来核心功能基本迁移完成后，再进行 CSV / JSON 拆分。
```

### GameConfig 仍是 JS 静态配置文件

```text
严重程度：低
问题：GameConfig 目前仍是 JS 静态配置文件，尚未拆分到 CSV / JSON。
影响：策划仍需要编辑 JS 文件调参，暂时不能通过表格或外部配置文件维护。
当前状态：这是当前阶段有意保留的中间形态。未来核心功能基本迁移完成后，再进行 CSV / JSON 拆分。
备注：GameConfig 字段虽然使用英文 key，但必须配套中文注释，便于策划理解和维护。
```
