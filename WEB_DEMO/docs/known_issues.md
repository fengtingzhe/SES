# Known Issues

## v0.6-refactor 仍不包含防御、人口、职业和特殊事件

严重程度：中

问题：
v0.6-refactor 只修复临时辉石主动拾回，并迁移矿山、采矿派工、持续产出和矿山占用释放。

当前不包含：
- 围墙、墙基、围墙 HP 和黑影攻墙。
- 弓箭手、弓箭手营、自动射击和完整职业系统。
- 流民、流民返程、待转职人口和人口补给。
- 工人自动恢复原任务的完整策略。
- 新营地建成后完整生成职业点、墙基等设施。
- 颠倒森林、狐嫁女、小地图、正式图片、音乐、字体或 CSV / JSON 配置读取。

影响：
当前版本可以验证辉石操作闭环与矿山资源补给，但还不能验证防御闭环、人口补给、职业转换或特殊事件。

处理方式：
后续 v0.7-refactor 起继续按任务卡迁移人口、职业、防御或特殊系统；未进入任务卡前不提前接入。

## v0.2-refactor 地图生成只是基础骨架

严重程度：中

问题：
当前地图生成保留 GPT_DEMO 的基础参数、主路径、分支、阻挡、水域骨架、起点、终点、少量辉石，并在 v0.3 增加可砍树、断桥和旧火塘。

影响：
当前地图不是完整 GPT_DEMO 地图生成迁移，尚未包含雾门、矿山、流民火堆、颠倒森林和狐嫁女；营地设施生成也仍是最小结构。

处理方式：
后续版本迁移对应系统时，再逐项恢复这些地图节点。

## v0.2-refactor 辉石放置尚未连接黑影诱敌（历史记录，已在 v0.4 解除）

严重程度：中

问题：
当前临时辉石拥有 10 秒生命周期，并消耗玩家辉石，但尚未连接黑影拾取、诱敌和玩家失败缓冲。

影响：
当前只能验证“无目标 Space 放置辉石”的基础交互，不能验证夜晚压力。

处理方式：
v0.4-refactor 已接入辉石诱敌和失败缓冲；后续防御、人口和工人夜晚风险仍按后续任务迁移。

## 旧 WEB_DEMO v0.2 - v0.6 路线已废弃

严重程度：高

问题：
旧 WEB_DEMO 路线是在重新开发一套类似 GPT_DEMO 的功能，而不是先理解并继承 GPT_DEMO 已验证的玩法规则。

影响：
如果继续沿用旧路线，GPT_DEMO 已经验证过的交互逻辑、系统关系和体验节奏可能无法被 WEB_DEMO 继承。

当前状态：
已决定废弃旧路线，并从 v0.1-refactor 重新开始 WEB_DEMO 阶段。

处理方式：
- 先提取 GPT_DEMO 规则基线。
- 再按规则基线重构 WEB_DEMO。
- 后续任务卡不得把旧 v0.2 - v0.6 的实现作为主线参考。

## GPT_DEMO 规则中仍有待确认项

严重程度：中

问题：
GPT_DEMO 中部分表现可能是原型简化，也可能是正式规则，需要策划确认后再迁移。

待确认项：
- 围墙是否继续可通行。
- 采矿派工是否保持 0 辉石成本。
- 地图尺寸是否严格继承 74 x 54。
- 工人威胁范围是否保持横纵差不超过 3，还是与黑影 4 格战术范围统一。
- F2 标签和 F3 小地图是正式功能还是开发辅助。
- 流民火堆是否无限刷新。
- 狐嫁女事件是否进入早期重构版本。
- 营地生成设施的相对位置是否严格继承。
- 一天 95 秒是否作为 WEB_DEMO 默认节奏锁定。
- 胜利结算是否只保留到达目标提示，还是需要完整结算界面。

当前状态：
已记录在 `WEB_DEMO/design/systems/gpt_demo_rule_baseline_v1.md` 和 `WEB_DEMO/design/audit/gpt_to_web_rule_audit_v1.md`。

处理方式：
在相关系统进入开发任务卡前确认；未确认前默认继承 GPT_DEMO 当前表现，不擅自重设计。
