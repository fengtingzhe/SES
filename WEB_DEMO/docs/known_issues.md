# Known Issues

## v0.2-refactor 只包含基础交互

严重程度：中

问题：
v0.2-refactor 只迁移最小工程骨架、基础地图、玩家移动、朝向、视野、Space 互动、辉石采集与放置。

当前不包含：
- 工人派遣、砍树、修桥和营地建设。
- 昼夜、黑影、雾门和失败缓冲。
- 围墙、弓箭手、矿山、流民和职业转换。
- 颠倒森林、狐嫁女、小地图、正式图片、音乐、字体或 CSV / JSON 配置读取。

影响：
当前版本只能验证基础手感和交互优先级，不能代表 GPT_DEMO 的完整玩法闭环。

处理方式：
后续从 v0.3-refactor 开始按规则基线逐步迁移工人、昼夜、黑影、防御、矿山和人口系统。

## v0.2-refactor 地图生成只是基础骨架

严重程度：中

问题：
当前地图生成只保留 GPT_DEMO 的基础参数、主路径、分支、阻挡、水域骨架、起点、终点和少量辉石。

影响：
当前地图不是完整 GPT_DEMO 地图生成迁移，尚未包含断桥、旧火塘、雾门、营地、矿山、流民火堆、颠倒森林和狐嫁女。

处理方式：
后续版本迁移对应系统时，再逐项恢复这些地图节点。

## v0.2-refactor 辉石放置尚未连接黑影诱敌

严重程度：中

问题：
当前临时辉石拥有 10 秒生命周期，并消耗玩家辉石，但尚未连接黑影拾取、诱敌和玩家失败缓冲。

影响：
当前只能验证“无目标 Space 放置辉石”的基础交互，不能验证夜晚压力。

处理方式：
在 v0.4-refactor 迁移黑影局部压力系统时接入辉石诱敌和失败缓冲。

## 旧 WEB_DEMO v0.2 - v0.6 路线已废弃

严重程度：高

问题：
旧 WEB_DEMO 路线是在重新开发一套类似 GPT_DEMO 的功能，而不是先理解并继承 GPT_DEMO 已验证的玩法规则。

影响：
如果继续沿用旧路线，GPT_DEMO 已经验证过的交互逻辑、系统关系和体验节奏可能无法被 WEB_DEMO 继承。

当前状态：
已决定废弃旧路线，并从 v0.1-refactor 重新开始 WEB_DEMO 阶段。

处理方式：
- 先提取 GPT_DEMO 规则基线。
- 再按规则基线重构 WEB_DEMO。
- 后续任务卡不得把旧 v0.2 - v0.6 的实现作为主线参考。

## v0.1 阶段 WEB_DEMO 暂不可运行（已在 v0.2 解除）

严重程度：中

问题：
v0.1-refactor 阶段不包含可运行游戏工程。

影响：
该问题在 v0.1-refactor 中导致无法执行 `npm run dev`，也不能试玩 WEB_DEMO。

当前状态：
已在 v0.2-refactor 中重新创建最小可运行工程骨架。

后续处理：
后续问题不再是“不可运行”，而是“只包含基础交互，完整玩法尚未迁移”。

## GPT_DEMO 规则中仍有待确认项

严重程度：中

问题：
GPT_DEMO 中部分表现可能是原型简化，也可能是正式规则，需要策划确认后再迁移。

待确认项：
- 围墙是否继续可通行。
- 采矿派工是否保持 0 辉石成本。
- 地图尺寸是否严格继承 74 x 54。
- 工人威胁范围是否保持横纵差不超过 3，还是与黑影 4 格战术范围统一。
- F2 标签和 F3 小地图是正式功能还是开发辅助。
- 流民火堆是否无限刷新。
- 狐嫁女事件是否进入早期重构版本。
- 营地生成设施的相对位置是否严格继承。
- 一天 95 秒是否作为 WEB_DEMO 默认节奏锁定。
- 胜利结算是否只保留到达目标提示，还是需要完整结算界面。

当前状态：
已记录在 `WEB_DEMO/design/systems/gpt_demo_rule_baseline_v1.md` 和 `WEB_DEMO/design/audit/gpt_to_web_rule_audit_v1.md`。

处理方式：
在相关系统进入开发任务卡前确认；未确认前默认继承 GPT_DEMO 当前表现，不擅自重设计。
