前置条件：

- 当前任务只修复 v0.4 小问题，并迁移工人夜晚感知、撤退、任务释放、lost 和黑影抓工人。

操作步骤：
1. 进入 `WEB_DEMO` 目录。
2. 执行 `npm install`。
3. 执行 `npm run dev`。
4. 打开浏览器访问 Vite 输出的本地地址。
5. 观察初始提示和页面标题，确认版本为 v0.5-refactor。
6. 等待夜晚或通过测试脚本模拟夜晚，确认黑影达到同场上限时暂停生成。
7. 观察黑影到达 raidCamp 附近后，不会在夜晚中无故立刻返回雾门。
8. 派工人执行砍树、修桥或建营地任务。
9. 夜晚让黑影接近正在执行任务的工人，观察工人进入撤退状态。
10. 确认原任务 reserved 被释放，后续可重新派工。
11. 尝试在撤退中重新派该工人，确认不会被选中。
12. 观察撤退工人回到所属家园后恢复为空闲。
13. 让黑影接触工人，确认工人进入 lost 状态，黑影消失。
14. 观察 HUD，确认能看到撤退 / 失踪相关统计。

预期结果：
1. 初始提示文案不再停留在 v0.3。
2. 黑影同场数量达到上限即暂停生成。
3. 黑影到达 raidCamp 附近后不会无故立刻返回雾门。
4. 工人夜晚能感知附近黑影。
5. 工人感知危险后进入撤退状态。
6. 撤退工人向所属家园移动。
7. 撤退工人不可被重新派工。
8. 工人撤退时释放当前任务 reserved。
9. 工人回到家园后恢复为空闲状态。
10. 黑影可以抓走工人。
11. 被抓工人进入 lost 状态。
12. lost 工人不再计入可用工人。
13. 工人被抓时释放当前任务 reserved。
14. HUD 能显示撤退 / lost 相关状态或统计。
15. 本轮没有迁移围墙、弓箭手、矿山、流民、职业系统或特殊事件。
16. GPT_DEMO 未被修改。
17. 审计文档记录 GPT_DEMO 到 WEB_DEMO 的迁移关系。

## v0.4-refactor 昼夜与黑影局部压力迁移验收

测试名称：WEB_DEMO v0.4-refactor 昼夜与黑影局部压力迁移

前置条件：
- 已安装 Node.js。
- 当前任务只迁移昼夜、雾门、夜晚黑影生成、家园推进目标、4 格局部感知、目标锁、辉石诱导、玩家辉石损失和失败状态。

操作步骤：
1. 进入 `WEB_DEMO` 目录。
2. 执行 `npm install`。
3. 执行 `npm run dev`。
4. 打开浏览器访问 Vite 输出的本地地址。
5. 观察 HUD，确认显示天数和白天 / 黄昏 / 夜晚阶段。
6. 在地图中确认至少存在 2 个雾门。
7. 等待进入夜晚，观察黑影只从雾门出现。
8. 持续观察同一夜晚，确认默认最多生成 2 个黑影，且不是同时全部出现。
9. 在黑影 4 格范围内放置临时辉石，确认黑影优先转向辉石。
10. 观察黑影接触辉石后，辉石被移除且黑影消失。
11. 让玩家持有辉石时接触黑影，确认玩家损失 1 个辉石且黑影消失。
12. 将玩家辉石消耗到 0 后再接触黑影，确认进入失败状态。
13. 失败后按 R，确认游戏可以重置。
14. 夜晚结束后观察场上黑影，确认其返回雾门或被回收。

预期结果：
1. 工程能正常启动。
2. 浏览器控制台无 JavaScript 报错。
3. HUD 显示天数和当前阶段。
4. 地图中至少存在 2 个雾门，WorldRenderer 能显示雾门。
5. 黑影只在夜晚从随机雾门生成。
6. 每夜默认最多生成 2 个黑影，生成间隔约为 3 秒。
7. 同场黑影超过 5 个时暂停生成。
8. 黑影无局部目标时向最近家园推进。
9. 黑影只响应 4 格战术范围内目标。
10. 黑影目标锁约 1.2 秒。
11. 黑影优先靠近 4 格范围内的临时辉石。
12. 黑影接触辉石后，辉石移除，黑影消失。
13. 黑影接触玩家后，若玩家有辉石，则玩家损失 1 个辉石，黑影消失。
14. 玩家 0 辉石时接触黑影，进入失败状态。
15. R 重置仍可用。
16. 夜晚结束后，黑影返回雾门或被回收。
17. 本轮没有迁移围墙、弓箭手、矿山、流民、工人夜晚避险或特殊事件。
18. GPT_DEMO 未被修改。
19. 审计文档记录 GPT_DEMO 到 WEB_DEMO 的迁移关系。
