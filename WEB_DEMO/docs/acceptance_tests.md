## v0.2 核心循环验收

```text
测试名称：WEB_DEMO v0.2 核心循环
前置条件：已安装 Node.js
操作步骤：
1. 进入 WEB_DEMO 目录
2. 执行 npm install
3. 执行 npm run dev
4. 打开浏览器
5. 使用 WASD / 方向键移动主角
6. 靠近辉石并确认 HUD 辉石数量增加
7. 在无互动目标时按 Space 放置辉石，确认地面出现辉石光点并会随时间消失
8. 靠近树障按 Space 派遣工人，等待砍树完成
9. 靠近断桥按 Space 派遣工人，等待修桥完成
10. 靠近旧营地按 Space 派遣工人，等待营地点亮
11. 观察 HUD 中第几天和昼夜阶段变化，夜晚画面变暗
12. 抵达远方信标
预期结果：
1. 浏览器控制台无 JavaScript 报错
2. 玩家可以移动
3. 小型地图可生成并包含部落、树障、河流、断桥、旧营地、目标点
4. 辉石拾取、放置和限时消失可用
5. 工人可执行 idle / moving / working / returning 状态流转
6. 砍树后树障变为可通行地面，并掉落 1 个辉石
7. 修桥后断桥变为可通行桥
8. 点亮营地后地图上有明显营火表现
9. 工人完成任务后返回部落或已激活营地
10. 抵达目标后显示“你抵达了阶段终点。”
11. WEB_DEMO/index.html 仍只加载 /src/main.js
12. WEB_DEMO/src/main.js 未堆积大量游戏逻辑
```

## v0.4 夜晚黑影压力系统验收

```text
测试名称：WEB_DEMO v0.4 夜晚黑影压力系统
前置条件：已安装 Node.js，v0.3 GameConfig 已完成
操作步骤：
1. 进入 WEB_DEMO 目录
2. 执行 npm install
3. 执行 npm run dev
4. 打开浏览器
5. 观察地图上是否存在黑雾口
6. 等待进入夜晚
7. 观察 HUD 中黑影计数
8. 在黑影附近放置辉石
9. 派遣工人外出执行砍树、修桥或点亮营地任务
10. 观察黑影接近工人时的结果
11. 如果工人被抓，重新靠近同一个任务目标并尝试再次派遣工人
12. 等待天亮
预期结果：
1. 浏览器控制台无 JavaScript 报错
2. GameConfig.js 存在 monster 配置段，关键字段有中文注释
3. 地图中存在黑雾口或等价黑影出生点
4. 夜晚能生成黑影，且每晚生成数量读取 GameConfig.monster.perNight
5. 黑影默认朝最近已点亮营地或起点部落移动
6. 黑影只在 GameConfig.monster.tacticalRange 范围内感知目标
7. 黑影优先追逐玩家放置的辉石，自然生成辉石不吸引黑影
8. 黑影接触放置辉石后，辉石消失，黑影被驱散
9. 黑影可抓走 moving / working / returning 状态的外出工人
10. 工人被抓后会从可用工人中移除
11. 工人被抓后原任务目标 reserved 状态被释放，可重新派遣工人
12. 天亮后残留黑影被清除或撤退
13. 本轮不包含围墙、弓箭手、防御战、矿山、流民或奇遇
```

## v0.5 围墙 / 弓箭手 / 防御系统验收

```text
测试名称：WEB_DEMO v0.5 围墙 / 弓箭手 / 防御系统
前置条件：已安装 Node.js，v0.4 黑影系统已完成
操作步骤：
1. 进入 WEB_DEMO 目录
2. 执行 npm install
3. 执行 npm run dev
4. 打开浏览器
5. 收集足够辉石
6. 靠近墙基或可建墙位置，按 Space 派工人建墙
7. 等待围墙建成
8. 靠近弓箭训练点，招募至少 1 名弓箭手
9. 等待夜晚黑影生成
10. 观察黑影接近围墙后的行为
11. 观察弓箭手是否自动攻击黑影
12. 如围墙被攻击，观察 HP 或受损表现
预期结果：
1. 浏览器控制台无 JavaScript 报错
2. GameConfig.js 存在 wall / archer 配置段，关键字段有中文注释
3. GameConfig.monster 包含黑影 HP、攻击围墙间隔、攻击围墙伤害和攻击围墙范围配置
4. 玩家可以消耗辉石派工人建造围墙
5. 围墙拥有 HP，并显示 HP 或受损表现
6. 黑影可以在局部感知范围内优先攻击围墙
7. 围墙 HP 归零后被摧毁或恢复为墙基
8. 玩家可以消耗辉石招募弓箭手
9. 弓箭手可以自动攻击射程内黑影
10. 黑影拥有 HP，可以被弓箭手击败
11. v0.4 的黑影夜晚压力、辉石诱敌、工人被抓仍可运行
12. 本轮不包含矿山、流民、奇遇或复杂职业系统
```
