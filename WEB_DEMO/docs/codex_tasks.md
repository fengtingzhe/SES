# 当前任务：WEB_DEMO v1.0-refactor 体验回归与辅助信息整理

## 任务背景

WEB_DEMO 当前采用新的重构路线：

```text
GPT_DEMO = 玩法验证原型与规则基线
WEB_DEMO = 工程化重构版本
```

当前已完成：

```text
v0.1-refactor：GPT_DEMO 规则基线提取与 WEB_DEMO 阶段重定义
v0.2-refactor：最小工程骨架与基础交互迁移
v0.3-refactor：自动拾取修正 + 工人派工与营地扩张迁移
v0.4-refactor：昼夜与黑影局部压力迁移
v0.5-refactor：v0.4 小修复 + 工人夜晚避险、被抓与任务释放
v0.6-refactor：临时辉石拾回修复 + 矿山与持续产出迁移
v0.7-refactor：流民、人口池与职业转换迁移
v0.7-audit-fix：审计文档同步修复
v0.8-refactor：围墙、防御与弓箭手战斗闭环迁移
v0.8-audit-fix：黑影攻击弓箭手范围偏差与围墙通行规则确认
v0.9-refactor：特殊事件与终点体验迁移
v0.9-audit-fix：补充 v0.9 代码迁移记录
```

WEB_DEMO 已经迁移了 GPT_DEMO 的主要玩法链路：探索、辉石、派工、营地、昼夜、黑影、工人风险、矿山、流民、职业、围墙、防御、特殊事件和终点目标。

v1.0 的目标不是新增大系统，而是做一次完整体验回归与辅助信息整理，让 WEB_DEMO 成为可以稳定试玩、便于后续继续迭代的阶段版本。

# 本轮只做

## 一、小地图 / 辅助信息回归

从 `GPT_DEMO/index.html` 中定位 F3 小地图或类似辅助信息逻辑。

必须继承信息价值：

```text
1. 能帮助玩家理解地图大致结构。
2. 能看出起点 / 家园 / 终点方向。
3. 能看出已探索区域或基础路线信息。
4. 不应完全破坏探索未知感。
```

v1.0 最小实现：

```text
1. 可以通过按键切换小地图，例如 F3 或 M。
2. 小地图可以显示已探索地块的简化点阵。
3. 至少标记玩家、起点 / 家园、终点。
4. 推荐标记营地、雾门、矿山、流民火堆、狐狸事件点、颠倒森林区域。
5. 如果不做完整小地图，也必须实现等价辅助信息面板，并在审计中说明为有意重构。
```

## 二、标签与交互提示整理

对照当前已实现系统，检查玩家靠近目标时的提示是否完整、准确、不会误导。

必须覆盖：

```text
1. 自然辉石自动拾取提示。
2. 临时辉石 Space 拾回提示。
3. 砍树、修桥、建营地、建墙提示。
4. 矿山派工 / 已占用 / 采矿中提示。
5. 流民火堆招募 / 冷却提示。
6. 工人屋和弓箭手营转职提示。
7. 雾门危险来源提示。
8. 颠倒森林方向反转提示。
9. 狐狸成亲事件开始 / 进行 / 成功 / 失败提示。
10. 终点灯塔 / 看海目标提示。
```

本轮只做文字和提示整理，不接入正式 UI 资源。

## 三、HUD 信息完整化

HUD 需要检查并整理当前所有核心状态。

建议至少显示：

```text
1. 版本号。
2. 当前天数和阶段。
3. 玩家辉石数量。
4. 当前状态：playing / failed / completed。
5. 空闲工人 / 总工人 / 逃跑工人 / lost 工人。
6. 采矿工人数。
7. 待转职人口。
8. 返程流民。
9. 弓箭手数量 / 瞄准 / 冷却。
10. 家园 / 营地数量。
11. 围墙数量 / 受损围墙数量。
12. 黑影数量 / 本夜生成数量。
13. 当前是否处于颠倒森林。
14. 狐狸婚仪状态。
15. 当前交互提示。
16. 小地图 / 辅助信息快捷键提示。
```

要求：

```text
1. 信息不要遮挡主画面。
2. 文案尽量统一。
3. 重要状态优先级更高，例如失败、胜利、黑夜、事件中。
```

## 四、规则一致性回归

对照 `GPT_DEMO/index.html` 和现有审计文档，做一次规则回归检查。

至少检查：

```text
1. 初始地图尺寸、起点、终点、初始辉石、初始工人。
2. 自然辉石自动拾取与临时辉石主动拾回。
3. 工人派工、reserved、不可取消、返程。
4. 砍树、修桥、建营地、建墙成本。
5. 昼夜、雾门、黑影生成数量、局部 4 格感知。
6. 黑影目标优先级。
7. 辉石诱敌。
8. 玩家受击掉辉石和 0 辉石失败。
9. 工人逃跑、lost、任务释放。
10. 矿山 0 成本派工、30 秒产出、占用释放。
11. 流民招募、返程、待转职人口、转职成本。
12. 围墙 HP、攻墙、弓箭手瞄准、冷却、伤害。
13. 颠倒森林、狐狸成亲、终点完成。
```

如果发现偏差：

```text
1. 小问题可直接修复。
2. 会影响策划决策的问题，写入 known_issues。
3. 会改变规则的调整，必须写入审计文档为有意重构或待确认。
```
