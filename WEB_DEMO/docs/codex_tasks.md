## 任务背景

WEB_DEMO 当前采用新的重构路线：

```text
GPT_DEMO = 玩法验证原型与规则基线
WEB_DEMO = 工程化重构版本
```

当前已完成：

```text
v0.1-refactor：GPT_DEMO 规则基线提取与 WEB_DEMO 阶段重定义
v0.2-refactor：最小工程骨架与基础交互迁移
v0.3-refactor：自动拾取修正 + 工人派工与营地扩张迁移
v0.4-refactor：昼夜与黑影局部压力迁移

```

v0.6 已经补齐了辉石资源操作闭环和矿山补给。下一步需要迁移人口补给与职业转换基础，为后续防御系统做准备。

## 本轮目标

从 `GPT_DEMO/index.html` 中定向迁移：

```text
流民火堆
招募流民
流民返程
待转职人口池
工人屋
弓箭手营
消耗辉石转职为工人 / 弓箭手
```

本轮重点验证：

```text
1. 地图中存在流民火堆。
2. 玩家可以消耗辉石招募流民。
3. 被招募的流民会返回最近家园。
4. 流民抵达家园后，不直接变工人，而是进入待转职人口池。
5. 营地附近存在工人屋和弓箭手营，或至少有最小职业转换点。
6. 玩家可以消耗辉石把待转职人口转为工人。
7. 玩家可以消耗辉石把待转职人口转为弓箭手。
8. HUD 能显示待转职人口、工人、弓箭手等基础人口信息。
```

# 本轮只做

## 一、流民火堆

从 `GPT_DEMO/index.html` 中定位流民火堆生成和刷新逻辑。

必须继承：

```text
1. 地图中存在流民火堆。
2. 流民火堆是人口补给来源。
3. 玩家与流民火堆互动可招募流民。
4. 招募后流民火堆进入冷却或刷新状态。
5. GPT_DEMO 中流民火堆约 10 秒冷却 / 刷新，本轮默认继承。
```

v0.7 最小实现：

```text
1. 地图中至少生成 1 个流民火堆。
2. WorldRenderer 能显示流民火堆。
3. InteractionSystem 能识别流民火堆互动。
```

---

## 二、招募流民

必须继承：

```text
1. 招募流民需要消耗 1 个辉石。
2. 玩家通过 Space 与流民火堆互动招募。
3. 如果辉石不足，应提示无法招募。
4. 招募后生成一个返程流民实体。
5. 招募后流民火堆进入冷却 / 空置状态。
```

---

## 三、流民返程

必须继承：

```text
1. 流民被招募后会返回最近家园。
2. 流民在返程途中不是立即可用工人。
3. 流民抵达家园后进入待转职人口池。
4. 流民抵达后从返程实体列表中移除。
```

本轮暂不实现：

```text
1. 黑影攻击返程流民。
2. 流民夜晚减速。
3. 流民被抓或 lost。
```

这些留到后续黑影目标扩展版本。

---

## 四、待转职人口池

必须继承：

```text
1. 流民抵达家园后进入待转职人口池。
2. 待转职人口不是工人，也不是弓箭手。
3. 待转职人口需要通过职业点消耗辉石转职。
```

建议状态：

```text
state.population.idle 或 state.population.unassigned
```

---

## 五、工人屋与弓箭手营

从 `GPT_DEMO/index.html` 中定位工人屋 / 弓箭手营相关代码。

必须继承：

```text
1. 工人屋用于把待转职人口转为工人。
2. 弓箭手营用于把待转职人口转为弓箭手。
3. 转职需要消耗 1 个辉石。
4. 转职需要有待转职人口。
5. 职业点应该与营地 / 家园结构关联。
```

v0.7 最小实现：

```text
1. 起点家园附近生成工人屋和弓箭手营。
2. 新营地建成后是否生成职业点可先按 GPT_DEMO 迁移；若复杂，可记录为待确认。
3. WorldRenderer 能显示工人屋和弓箭手营。
4. InteractionSystem 能识别职业点互动。
```
