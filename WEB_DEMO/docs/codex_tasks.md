# 当前任务：WEB_DEMO v0.8-refactor 围墙、防御与弓箭手战斗闭环迁移

## 任务背景

WEB_DEMO 当前采用新的重构路线：

```text
GPT_DEMO = 玩法验证原型与规则基线
WEB_DEMO = 工程化重构版本
```

当前已完成：

```text
v0.1-refactor：GPT_DEMO 规则基线提取与 WEB_DEMO 阶段重定义
v0.2-refactor：最小工程骨架与基础交互迁移
v0.3-refactor：自动拾取修正 + 工人派工与营地扩张迁移
v0.4-refactor：昼夜与黑影局部压力迁移
v0.5-refactor：v0.4 小修复 + 工人夜晚避险、被抓与任务释放
v0.6-refactor：临时辉石拾回修复 + 矿山与持续产出迁移
v0.7-refactor：流民、人口池与职业转换迁移
v0.7-audit-fix：审计文档同步修复
```

v0.7 已经补齐人口补给和职业转换基础。当前弓箭手已经可以被转职出来，但还没有战斗能力；围墙系统也尚未迁移。

v0.8 的目标是迁移第一版防御闭环。

## 本轮目标

从 `GPT_DEMO/index.html` 中定向迁移：

```text
墙基
建造围墙
围墙 HP
黑影局部目标中的围墙优先级
黑影攻击围墙
弓箭手夜晚自动瞄准黑影
弓箭手攻击 / 击杀黑影
弓箭手冷却
防御相关 HUD 和表现
```

本轮重点验证：

```text
1. 地图中存在墙基。
2. 玩家可以消耗辉石派工人建墙。
3. 建成后墙基变成围墙。
4. 围墙有 HP，默认 HP 3。
5. 黑影局部目标中，围墙优先级高于工人、弓箭手、玩家，但低于辉石。
6. 黑影接触围墙后攻击围墙，而不是直接穿过防线找玩家。
7. 围墙被攻击后 HP 下降，HP 归零后损坏或退回墙基状态。
8. 弓箭手夜晚能自动瞄准范围内黑影。
9. 弓箭手攻击黑影造成 1 点伤害。
10. 黑影 HP 为 1，因此被命中后消失。
11. 弓箭手有冷却时间，默认约 2.2 秒。
12. 弓箭手瞄准时间默认约 0.6 秒。
```

# 本轮只做

## 一、墙基与围墙地块

从 `GPT_DEMO/index.html` 中定位墙基、围墙相关地块和生成逻辑。

必须继承：

```text
1. 地图中存在墙基。
2. 墙基是可建设防御点。
3. 建成后变成围墙。
4. 围墙默认 HP 3。
5. 围墙作为黑影防御目标。
```

v0.8 最小实现：

```text
1. 起点家园附近至少生成 1 - 2 个墙基。
2. 新营地是否生成墙基可按 GPT_DEMO 迁移；若复杂，可记录为待确认。
3. WorldRenderer 能显示墙基、围墙和围墙 HP。
```

---

## 二、派工建墙

必须继承：

```text
1. 建墙需要消耗 2 个辉石。
2. 玩家通过 Space 与墙基互动派工建墙。
3. 只有可用工人能执行建墙。
4. 墙基被派工后进入 reserved 状态，避免重复派工。
5. 工人到达后工作一段时间。
6. 建墙完成后，墙基变成围墙，HP 设为 3。
7. 工人完成后返回所属家园。
```

需要复用：

```text
v0.3 的派工、工作、返程、reserved 机制。
v0.5 的逃跑 / lost 时释放任务机制。
```

---

## 三、黑影攻击围墙

从 `GPT_DEMO/index.html` 中定位黑影攻击围墙逻辑。

必须继承：

```text
1. 黑影局部目标中，辉石优先级最高。
2. 围墙优先级高于工人、弓箭手、返程流民、玩家。
3. 黑影只响应 4 格范围内的围墙。
4. 黑影接触围墙后攻击围墙。
5. 黑影对围墙造成持续伤害，GPT_DEMO 表现为约每秒攻击。
6. 围墙 HP 归零后损坏或退回墙基状态。
```

v0.8 暂不要求：

```text
1. 营地损坏。
2. 围墙升级。
3. 多级防御。
```

## 四、弓箭手战斗

从 `GPT_DEMO/index.html` 中定位弓箭手夜晚防御逻辑。

必须继承：

```text
1. 弓箭手只在夜晚参与防御。
2. 弓箭手自动寻找射程内黑影。
3. 弓箭手默认瞄准约 0.6 秒。
4. 弓箭手攻击造成 1 点伤害。
5. 黑影 HP 为 1，因此被命中后消失。
6. 弓箭手攻击后进入约 2.2 秒冷却。
7. 弓箭手目标应优先选择距离较近或 GPT_DEMO 中实际使用的目标。
```

建议实现：

```text
1. 新增 ArcherSystem.js，或扩展已有单位系统。
2. state.archers 中补充 aimingTargetId、aimTimer、cooldown 等字段。
3. GameConfig.archer 中配置 range、aimSeconds、cooldownSeconds、damage。
4. WorldRenderer 显示弓箭手瞄准 / 攻击状态。
```

# 验收标准

必须满足：

```text
1. 地图中至少存在 1 个墙基。
2. 玩家可以消耗 2 个辉石派工人建墙。
3. 建墙任务 reserved 防止重复派工。

5. 围墙默认 HP 为 3。
6. 黑影 4 格范围内能识别围墙。
7. 黑影目标优先级为：辉石 > 围墙 > 工人 > 弓箭手 > 玩家 > 家园推进。
8. 黑影接触围墙后攻击围墙。
9. 围墙 HP 会被黑影攻击降低。
10. 围墙 HP 归零后损坏或退回墙基状态。
11. 弓箭手夜晚自动寻找射程内黑影。
12. 弓箭手瞄准约 0.6 秒后攻击。
13. 弓箭手攻击造成 1 点伤害。
14. 黑影 HP 为 1，被弓箭手命中后消失。
15. 弓箭手攻击后进入约 2.2 秒冷却。
16. HUD 或表现能显示围墙 / 弓箭手战斗相关状态。
17. 本轮没有迁移流民被抓、流民夜晚减速、特殊事件、小地图或正式资源。
18. GPT_DEMO 未被修改。
19. 审计文档记录 GPT_DEMO 到 WEB_DEMO 的迁移关系。
```
