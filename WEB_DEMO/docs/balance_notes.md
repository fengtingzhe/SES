# WEB_DEMO v1.3-dev-balance-console 数值基线记录

本文档记录当前试玩数值的基线、节奏假设和后续调参观察点。本轮只整理与记录，不新增正式玩法，不接入 JSON / CSV / 存档，也不修改 GPT_DEMO。

## 1. 数值目标

- 当前版本目标：让 WEB_DEMO 可以快速验证“探索、采集、派工、夜晚压力、防御、人口补给、特殊事件、终点目标”的完整试玩节奏。
- 当前数值不是正式平衡，仅作为后续策划调参的对照基线。
- 本轮未主动改变正式玩法数值；代码层只修正随机辉石生成范围公式，避免边距语义错误导致分布过宽。

## 2. Demo 目标时长与旅程阶段

当前建议把一局完整 WEB_DEMO 试玩目标先记录为 10 到 15 分钟。这个目标用于后续调参对照，不代表正式时长已经锁定。

| 阶段 | 预计时间 | 玩家目标 | 核心压力 | 主要资源变化 |
| --- | ---: | --- | --- | --- |
| 早期 | 0 到 3 分钟 | 熟悉移动、拾取辉石、派工砍树 / 修桥 | 辉石是否够用、路线是否清晰 | 开局 6 辉石 + 起点与主路补给，支出集中在派工 |
| 中期 | 3 到 8 分钟 | 建营地、采矿、招募流民、形成待转职池 | 夜晚黑影、工人撤退、矿山复工 | 矿山开始提供持续回流，流民 / 转职增加支出 |
| 后期 | 8 到 15 分钟 | 建墙、补弓箭手、处理特殊事件、抵达终点 | 防御是否足够、探索是否被天气 / 黑影打断 | 围墙与弓箭手消耗辉石，终点和事件提供完成目标 |

## 7. 地图资源分布基线

开局辉石包括：

- 起点旁固定辉石：保证进入游戏后可快速确认自动拾取。
- 主路固定辉石：保证推进路线中有稳定补给点。
- 随机辉石：提供轻随机探索奖励。

v1.3 修正后的随机辉石范围公式：

- x 范围：`randomMinX <= x < map.width - randomMaxXMargin`
- y 范围：`randomMinY <= y < map.height - randomMaxYMargin`
- `randomMaxXMargin` 表示右侧安全边距，不是随机范围长度。
- `randomMaxYMargin` 表示下侧安全边距，不是随机范围长度。

当前默认值下：

- x 范围为 `[6, 62)`。
- y 范围为 `[5, 44)`。
- 随机辉石尝试次数为 16 次，不保证最终生成 16 个，因为只有落在地面格时才会成功。

## 9. 试玩节奏参考

建议用 10 到 15 分钟完成一次完整回归试玩：

1. 开局 0 到 30 秒：确认自然辉石自动拾取、Space 放置临时辉石、工人派工。
2. 第一天白天：砍树、修桥、建营地，观察资源是否足够支撑至少两次关键建设。
3. 第一天黄昏到黑夜：测试临时辉石诱敌、黑影局部感知、玩家辉石损失。
4. 第二天：测试矿山持续产出、流民招募、待转职人口和职业转换。
5. 防御阶段：测试建墙、墙 HP、黑影攻击围墙、弓箭手自动射击。
6. 后段探索：测试颠倒森林、狐狸成亲、终点灯塔 / 看海目标。

## 10. Dev Console 调参用途

v1.3 新增内置 Dev Console，用于降低人工等待成本：

- F1 打开 / 关闭。
- 资源页用于辉石库存和地图辉石验证。
- 时间天气页用于快速切白天、黄昏、黑夜和触发天气。
- 单位页用于生成工人、弓箭手、黑影和召回工人。
- 事件页用于生成流民火堆、雾门、矿山、颠倒森林和狐狸成亲事件。

Dev Console 只服务开发与策划测试，不视为正式玩家玩法，不进入正式资源、存档或外部配置流程。

## 11. 后续观察点

- 6 个开局辉石是否让第一天建设过于宽松。
- 采矿 30 秒产出 1 个辉石是否足以支撑第二天以后扩张。
- 每夜 2 个黑影是否过轻，是否需要随天数或营地数量成长。
- 围墙 3 HP 与弓箭手一箭击杀是否让防御过强。
- 流民火堆是否需要生成次数或地图总量限制。
- 天气事件概率是否应该与区域、天数或当前资源压力挂钩。
