# GPT_DEMO：ChatGPT 快速可试玩草图阶段

## 定位

`GPT_DEMO/` 是正式 `WEB_DEMO/` 之前的快速验证区。

它用于支持制作人与 ChatGPT 在方案讨论阶段，快速生成可运行网页草图，让用户先直观看到方向是否一致。

核心目标不是工程质量，而是：

```text
快速看见；
快速试玩；
快速暴露偏差；
快速修正方向。
```

---

## 当前最新版本

```text
当前文件：GPT_DEMO/index.html
当前版本：gpt.6.2
当前状态：可试玩草图阶段
```

当前 Demo 用于验证：

```text
探索 → 收集辉石 → 派工人开路 → 工人沿路执行 → 夜晚怪物风险 → 营火/部落安全点 → 抵达阶段目标
```

---

## 如何运行

拉取仓库后，直接双击打开：

```text
GPT_DEMO/index.html
```

操作方式：

```text
WASD / 方向键：移动主角
鼠标点击树障：派工人砍树，消耗 1 辉石
鼠标点击断桥：派工人修桥，消耗 2 辉石
鼠标点击空地：派工人建营火，消耗 2 辉石
Space：投掷辉石光圈
R：重新生成 / 重新开始
```

---

## 与 WEB_DEMO 的区别

| 目录 | 作用 | 质量要求 |
|---|---|---|
| `GPT_DEMO/` | 方案讨论阶段的快速可试玩草图 | 可以粗糙，可以硬编码，重点是直观 |
| `WEB_DEMO/` | 正式 Web Demo 原型工作区 | 需要任务卡、结构清晰、可维护、可配置 |

# 版本记录

## gpt.1：初版可试玩草图

### 修改内容

- 新增 `GPT_DEMO/index.html` 单文件 HTML Demo。
- 实现 2D 45 度俯视角 / 等距视角基础画面。
- 实现主角移动探索。
- 实现小型随机森林地图。
- 实现辉石收集与消耗。
- 实现派工人砍树。
- 实现派工人修桥。
- 实现派工人建营火。
- 实现工人命令不可取消。
- 实现日夜循环。
- 实现夜晚黑林影追逐安全圈外工人。
- 实现部落、营火、辉石光圈安全区。
- 实现阶段目标：抵达蓝色信标。

### 修改原因

用于快速验证《看海去》的最小核心循环：

```text
探索 → 辉石分配 → 派工人开路 → 夜晚风险 → 抵达阶段目标
```

### 遗留问题

- 地图较小，远征感不足。
- 工人移动为直线，尚未处理复杂地形路径。
- 怪物逻辑较粗糙。

## gpt.2：地图扩大与性能优化

### 修改内容

- 地图从 `26×22` 扩大到 `52×44`。
- 地图面积约扩大为原来的 4 倍。
- 增加视口裁剪，只绘制屏幕附近地块。
- 增加小地图。
- 增加第二条河流，让远征距离更明显。
- 面板显示当前每帧绘制地块数量，方便观察性能。

### 修改原因

增强远征感、迷失感和距离感，同时保证大地图下浏览器 Canvas 不明显卡顿。

### 遗留问题

- 大地图下玩家引导仍较弱。
- 地图节奏仍由随机生成主导，尚未精确控制营火/风险节点。

---

## gpt.3：玩家受击与工人返回机制

### 修改内容

- 黑林影会攻击玩家。
- 玩家夜晚在安全圈外时可能被黑林影追击。
- 玩家被攻击时，如果身上有辉石，会随机掉落 1–3 个辉石。
- 掉落的辉石会散落在玩家附近，可重新捡回。
- 玩家辉石为 0 时，再次被攻击则游戏结束。
- 玩家受击后增加短暂无敌时间。
- 工人完成砍树、修桥、建营火后，不再立即空闲。
- 工人完成任务后会返回最近营火或部落。
- 工人抵达安全点后才重新变为空闲。
- HUD 增加“返回中”工人数量显示。

### 修改原因

让辉石同时承担资源与生命缓冲的作用，并让工人规划更接近《Kingdom New Lands》的风险结构。

### 遗留问题

- 工人返回路径仍有问题。
- 工人仍可能直线穿越不可通行地形。

---

## gpt.4：最近工人选择与寻路

### 修改内容

- 派工时不再默认选择第一个空闲工人。
- 系统计算所有空闲工人到任务点的可通行路径。
- 选择路径最短的空闲工人出发。
- 新增 BFS 网格寻路。
- 工人沿网格路径移动。
- 工人不再直线穿过森林、河流、岩石。
- 砍树 / 修桥时，工人会走到目标旁边的可通行格再工作。
- 完成任务后，工人也会沿可通行路径返回最近营火 / 部落。
- 增加淡黄色路径线，便于观察工人寻路。

### 修改原因

修正工人穿越不可通行地形的问题，并让派工选择更符合策略直觉。

### 遗留问题

- 返回中的工人不参与新任务距离计算，导致调度不自然。

---

## gpt.5：返回中工人可重新调度

### 修改内容

- `returning` 返回中的工人也会参与任务距离计算。
- 系统会在 `idle + returning` 工人中，选择到任务目标可通行路径最短的工人。
- 如果返回中的工人被选中，会中断返回路径，改为前往新任务。
- `moving / working / lost` 工人仍不参与新任务调度。

### 修改原因

修正工人完成任务后正在返回时，明明离新任务最近却不能被重新派遣的问题。

### 遗留问题

- 辉石拾取价值存在表现歧义：自然生成辉石和掉落辉石视觉相同，但数值不同。

---

## gpt.6：辉石簇与单颗掉落辉石

### 修改内容

- 明确规则：一颗辉石永远等于 1。
- 自然生成资源点改为“辉石簇”。
- 自然辉石簇视觉上显示为两颗辉石。
- 自然辉石簇拾取后获得 `+2`。
- 玩家受击掉落的辉石显示为单颗散落辉石。
- 单颗掉落辉石拾取后获得 `+1`。
- Toast 提示区分：
  - `获得辉石簇 +2`
  - `捡回掉落辉石 +1`

### 修改原因

解决“视觉上都是一颗辉石，但有时 +2、有时 +1”的规则歧义。

### 遗留问题

- 本次压缩代码时引入变量命名冲突，导致页面无法打开。

---

## gpt.6.1：修复页面打不开问题

### 修改内容

- 修复 gpt.6 页面无法打开的问题。
- 将容易冲突的变量命名拆开。
- `toast` 元素不再使用容易冲突的 `t`。
- 游戏时间变量改为 `gt`。
- 保留 gpt.6 的辉石规则：
  - 两颗辉石簇：`+2`
  - 单颗掉落辉石：`+1`

### 修改原因

gpt.6 中页面脚本启动失败，原因是变量 `t` 与 DOM 元素引用产生命名冲突。

### 遗留问题

- 砍树完成后树不消失。
- 修桥完成后桥不可通过。

---

## gpt.6.2：修复完成任务坐标反置问题

### 修改内容

- 修复完成任务时目标地块坐标写反的问题。
- 将：

```js
cell(w.job.ty, w.job.tx)
```

修正为：

```js
cell(w.job.tx, w.job.ty)
```

- 砍树完成后，目标树障会正确变为草地。
- 修桥完成后，断桥会正确变为可通行桥。
- 页面版本显示更新为 `gpt.6.2`。

### 修改原因

`cell(x, y)` 的参数顺序是 `x, y`，但旧代码把 `ty, tx` 传入，导致修改到了错误地块。

### 遗留问题

- 怪物与辉石逻辑仍待重构。
- 当前仍保留 Space 投掷辉石光圈逻辑，但后续方向倾向弱化或移除。
- 当前营火仍可在任意空地建造，后续方向倾向改为只能在固定“旧火塘 / 石环 / 灰烬遗址”建造。

# 已讨论但尚未写入 Demo 的方向

以下内容仅为玩法讨论结论，尚未进入 `GPT_DEMO/index.html`：

## 1. 怪物抢夺辉石

倾向规则：

```text
怪物攻击玩家 / 工人后，使目标掉落辉石。
怪物优先捡地上的辉石。
怪物捡到辉石后撤退。
玩家或工人无辉石再被攻击，才触发失败 / 失踪。
```

设计目的：

```text
让辉石成为可被抢夺的远征资源，而不是单纯生命值或货币。
```

---

## 2. 地上辉石不再自动形成保护圈

倾向规则：

```text
掉落在地上的辉石只是资源，不自动产生保护区。
怪物可以去捡地上的辉石。
```

设计原因：

```text
避免玩家通过高频“扔辉石 → 保护工人 → 捡辉石 → 再扔辉石”的方式，把游戏变成操作型救场玩法。
```

---

## 3. 弱化或移除 Space 投掷辉石光圈

当前 Demo 中仍保留 Space 投掷辉石生成临时光圈。

后续倾向：

```text
安全机制主要交给营火和部落。
辉石不再作为随手投掷的即时护盾。
```

设计方向：

```text
《看海去》更偏“事前规划型轻策略”，而不是“即时操作保护工人”。
```

---

## 4. 营火只能在固定地点建造

当前 Demo 中，营火仍可在任意已探索空地建造。

后续倾向：

```text
地图预生成若干营火遗址 / 旧火塘 / 石环 / 灰烬遗址。
玩家只能在这些固定点消耗辉石点亮营火。
```

设计目的：

```text
控制游戏节奏；
避免玩家走几步就建营火导致辉石耗尽；
让安全点成为路线规划中的战略节点。
```

# 当前玩法方向一句话总结

```text
《看海去》是远征规划型轻策略游戏：辉石是可被抢夺的远征资源，工人是带着资源出发的风险单位，怪物制造夜晚资源压力，营火与部落是稳定安全节点，玩家主要在行动前规划路线与风险，而不是靠高频操作救场。
```
