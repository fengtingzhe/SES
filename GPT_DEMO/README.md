# GPT_DEMO：ChatGPT 快速可试玩草图阶段

## 定位

`GPT_DEMO/` 是正式 `WEB_DEMO/` 之前的快速验证区。

它用于支持制作人与 ChatGPT 在方案讨论阶段，快速生成可运行网页草图，让用户先直观看到方向是否一致。

核心目标不是工程质量，而是：

```text
快速看见；
快速试玩；
快速暴露偏差；
快速修正方向。
```

---

## 当前最新版本

```text
当前文件：GPT_DEMO/index.html
当前版本：gpt.8
当前状态：可试玩草图阶段
```

当前 Demo 用于验证：

```text
移动探索 → Space 近距离互动 → 收集辉石 → 派工人开路 → 点亮固定营地 → 固定黑雾口夜晚风险 → 营地矿山采矿 → 抵达阶段目标
```

---

## 如何运行

拉取仓库后，直接双击打开：

```text
GPT_DEMO/index.html
```

PC 操作方式：

```text
WASD / 方向键：移动主角
Space：与附近目标互动
R：重新生成 / 重新开始
```

移动端方向，暂未实现：

```text
横屏
左侧虚拟摇杆：上下左右移动
右侧点击 / 按钮：互动
```

---

## 与 WEB_DEMO 的区别

| 目录 | 作用 | 质量要求 |
|---|---|---|
| `GPT_DEMO/` | 方案讨论阶段的快速可试玩草图 | 可以粗糙，可以硬编码，重点是直观 |
| `WEB_DEMO/` | 正式 Web Demo 原型工作区 | 需要任务卡、结构清晰、可维护、可配置 |

## gpt.3：玩家受击与工人返回机制

- 黑林影会攻击玩家。
- 玩家被攻击时，如果身上有辉石，会随机掉落 1–3 个辉石。
- 玩家辉石为 0 时，再次被攻击则游戏结束。
- 工人完成砍树、修桥、建营火后，会返回最近营火或部落，抵达安全点后才重新空闲。

---

## gpt.4：最近工人选择与寻路

- 派工时计算所有空闲工人到任务点的可通行路径。
- 选择路径最短的空闲工人出发。
- 新增 BFS 网格寻路。
- 工人不再直线穿过森林、河流、岩石。
- 增加淡黄色路径线，便于观察工人寻路。

---

## gpt.5：返回中工人可重新调度

- `returning` 返回中的工人也会参与任务距离计算。
- 系统会在 `idle + returning` 工人中，选择到任务目标可通行路径最短的工人。
- 如果返回中的工人被选中，会中断返回路径，改为前往新任务。

- 玩家受击掉落的辉石显示为单颗散落辉石，拾取 `+1`。

## gpt.8：固定黑雾口与营地矿山经营

### 修改内容

- 地图生成固定“黑雾口”。
- 夜晚怪物只从黑雾口出现，不再从目标附近随机生成。
- 白天黑雾口作为危险提示存在。
- 怪物继续遵循抢夺辉石逻辑：优先抢散落辉石，其次威胁安全区外的玩家和工人。
- 点亮旧火塘后，会在附近生成一个矿山。
- 玩家靠近矿山按 Space，可派最近可调度工人去采矿。
- 工人进入矿山后变为 `mining` 状态。
- `mining` 工人不参与砍树、修桥、点亮旧火塘等普通任务调度。
- 矿山每 10 秒产出 1 个单颗辉石。
- 玩家靠近产出的辉石后按 Space 拾取。
- HUD 增加黑雾口数量、当前怪物数量和采矿工人数显示。

### 修改原因

让夜晚风险从“随机出现”变为“可观察、可预判、可规划”的固定危险源；同时让营地从单纯安全点扩展为资源恢复点，验证“白天远征、夜晚经营”的双循环雏形。

### 遗留问题

- 矿山工人暂不支持召回。
- 矿山没有产出上限。
- 黑雾口位置和数量仍需试玩调优。
- 怪物返回黑雾口的动画暂未实现，当前捡到辉石后直接消失。
- 移动端虚拟摇杆和右侧互动按钮尚未实现。
